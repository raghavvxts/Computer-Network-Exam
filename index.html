CRC — Cyclic Redundancy Check
4 problems
C1
The CRC procedure — complete method, step by step
CONCEPT
▼
Given: Dataword M (k bits) and Generator P (degree r, so P has r+1 bits). r = len(P) − 1.
Step 1 — Append r zeros to M → M' = M followed by r zeros. M' has k+r bits.
Step 2 — Modulo-2 divide M' ÷ P using XOR long division (no borrows, no carries). XOR rule: 1⊕1=0, 0⊕0=0, 1⊕0=1.
Step 3 — Remainder = CRC. It will be exactly r bits. Pad with leading zeros if shorter.
Step 4 — Transmitted codeword = M + CRC (replace the r zeros with the CRC).
Verification: Receiver divides received frame ÷ P. Remainder = 0 → no error. Remainder ≠ 0 → error detected.
Division rule: If MSB of current chunk = 1 → XOR with P. If MSB = 0 → XOR with all-zeros same length (effectively just bring down next bit). Keep going until all bits of M' are consumed.

HAMMING DISTANCE & BLOCK CODING
📐
Hamming Distance & Block Coding
6 problems
H1
Find d(10101, 11110) and d(000, 011).
SOLVED
▼
Method: XOR the two words, then count the number of 1s in the result.
10101 XOR 11110 = 01011 → three 1s → d = 3
000 XOR 011 = 011 → two 1s → d = 2
H2
Find dmin for the set {00000, 01011, 10101, 11110}.
SOLVED
▼
Calculate all pairwise Hamming distances:
d(00000, 01011) = XOR = 01011 → 3
d(00000, 10101) = XOR = 10101 → 3
d(00000, 11110) = XOR = 11110 → 4
d(01011, 10101) = XOR = 11110 → 4
d(01011, 11110) = XOR = 10101 → 3
d(10101, 11110) = XOR = 01011 → 3
Answer
d_min = 3 (smallest value in the list)
H3
dmin = 4. What is the max number of errors this code can detect? Correct?
SOLVED
▼
Error Detection
s ≤ d_min − 1
Error Correction
t ≤ (d_min − 1) / 2
Detection: s = d_min − 1 = 4 − 1 = 3 errors detectable
Correction: t = floor((4−1)/2) = floor(1.5) = 1 error correctable
Key insight: d_min = 4 is even — part of correction capability is wasted. Error correction codes should ideally have odd d_min (3, 5, 7...). With d_min=4 you correct 1 but detect up to 3.
H4
C(n,k) coding scheme: k=4, r=3. How many total codewords? How many valid? How many unused?
SOLVED
▼
n = k + r = 4 + 3 = 7 bits per codeword
Total possible codewords: 2n = 27 = 128
Valid datawords (= valid codewords): 2k = 24 = 16
Unused codewords: 2n − 2k = 128 − 16 = 112
These 112 unused codewords are what make error detection possible — a corrupted codeword lands in an unused slot and is recognized as an error.
H5
What is d_min needed to detect up to 3 errors? To correct up to 2 errors?
PRACTICE
▼
To detect s=3 errors: d_min ≥ s + 1 = 3 + 1 = 4
To correct t=2 errors: d_min ≥ 2t + 1 = 2(2) + 1 = 5
H6
Codeword 00000 is sent. 01101 is received. How many bits are in error? What is the Hamming distance?
PRACTICE
▼
00000 XOR 01101 = 01101 → three 1s
Answer
3 bits in error. d(00000, 01101) = 3

SW2
Bandwidth = 1.5 Mbps, RTT = 45 ms, Packet size = 1 KB. Find link utilization.
SOLVED
▼
L = 1 KB = 1 × 1024 × 8 = 8192 bits
Tt = L/B = 8192 / (1.5 × 10⁶) = 0.005461 s = 5.461 ms
Tp = RTT/2 = 45/2 = 22.5 ms
a = Tp/Tt = 22.5 / 5.461 = 4.12
η = 1 / (1 + 2×4.12) = 1 / 9.24 = 0.108 = 10.8%
Answer
Link utilization ≈ 10.8%
SW3
Bit rate = 4 Kbps, Propagation delay = 20 ms, S&W protocol. Find minimum packet size for ≥20% efficiency.
SOLVED
▼
η ≥ 0.20 → 1/(1+2a) ≥ 1/5 → 1+2a ≤ 5 → 2a ≤ 4 → a ≤ 2
a = Tp/Tt ≤ 2 → Tp ≤ 2 × Tt
Tt = L/B → 0.020 ≤ 2 × (L / 4000) → L ≥ 0.020 × 4000 / 2 = 40 bits
Answer
Minimum packet size L ≥ 40 bits = 5 bytes
SW4
A 1 Mbps satellite link, altitude = 36,504 km, signal speed = 3×10⁸ m/s. Find propagation delay and check if S&W is efficient.
SOLVED
▼
Tp = D/V = 36,504,000 m / (3×10⁸ m/s) = 0.1217 s = 121.7 ms
For a 1000-bit packet: Tt = 1000 / 10⁶ = 1 ms
a = 121.7 / 1 = 121.7 → η = 1/(1+243.4) ≈ 0.41%
Answer
Tp ≈ 121.7 ms. S&W is extremely inefficient (0.41%) — sliding window protocols are needed for satellite links.

GS4
A protocol uses a 3-bit sequence number field. What is the max window size for Go-Back-N? For Selective Repeat?
PRACTICE
▼
3-bit sequence field → 2³ = 8 possible sequence numbers (0 to 7)
Go-Back-N max window = 2ⁿ − 1 = 8 − 1 = 7
Selective Repeat max window = 2^(n−1) = 2² = 4

SUBNETTING & CIDR
🌐
Subnetting & CIDR Addressing
6 problems
SN1
Subnetting method — complete procedure
CONCEPT
▼
Step 1: n = prefix length. Host bits = 32 − n.
Step 2 — Subnet Mask: n ones followed by (32-n) zeros. Last octet shortcuts: /25→128, /26→192, /27→224, /28→240, /29→248, /30→252.
Step 3 — First address (Network): AND IP with mask. OR: set last (32-n) bits of last relevant octet to 0.
Step 4 — Last address (Broadcast): OR IP with inverted mask. OR: set last (32-n) bits to 1.
Step 5: Total addresses = 2^(32-n). Usable hosts = 2^(32-n) − 2.
Subnetting a network: Bits borrowed = new prefix − original prefix. Number of subnets = 2^(borrowed bits).
SN2
IP = 205.16.37.39/28. Find: mask, first address, last address, total hosts.
SOLVED
▼
/28 → host bits = 4. Mask = 255.255.255.240
39 in binary = 00100111. Set last 4 bits to 0 → 00100000 = 32 → First = 205.16.37.32
Set last 4 bits to 1 → 00101111 = 47 → Last = 205.16.37.47
Total = 2⁴ = 16. Usable hosts = 14.
Answer
Network: 205.16.37.32 | Broadcast: 205.16.37.47 | Usable: 14 hosts
SN3
IP = 192.168.10.65/26. Find all subnet details.
SOLVED
▼
/26 → host bits = 6. Mask = 255.255.255.192
65 = 01000001. Top 2 bits (network within octet) = 01. Set last 6 bits to 0 → 01000000 = 64 → Network = 192.168.10.64
Set last 6 to 1 → 01111111 = 127 → Broadcast = 192.168.10.127
Host range: 192.168.10.65 → 192.168.10.126. Usable = 2⁶ − 2 = 62 hosts.
SN4
IP = 192.168.1.130/26. Find network address, broadcast, host range, usable hosts.
PRACTICE
▼
/26 → host bits = 6. Mask = 255.255.255.192
130 = 10000010. Top 2 network bits = 10. Set last 6 to 0 → 10000000 = 128 → Network = 192.168.1.128
Set last 6 to 1 → 10111111 = 191 → Broadcast = 192.168.1.191
Host range: .129 → .190. Usable = 62 hosts.
SN5
Network 172.16.0.0/16. Need subnets supporting ≥500 hosts each. Find new prefix and number of subnets.
SOLVED

BR2
Bit rate = 4000 bps, baud rate = 1000 baud. How many bits per signal element? How many levels?
SOLVED
▼
Bits per element = N/S = 4000/1000 = 4 bits per baud
Levels L: 4 = log₂(L) → L = 2⁴ = 16 signal levels
BR3
Bit rate = 2000 bps. Find baud rate for NRZ-L (worst case) and Manchester.
SOLVED
▼
NRZ-L worst case: c=1, r=1 → S = cN/r = 1×2000/1 = 2000 baud
Manchester: c=1, r=1/2 → S = 1×2000/(1/2) = 4000 baud
Manchester always needs double the baud rate of NRZ — that's the bandwidth penalty for built-in clock synchronisation.
BR4
Channel bandwidth = 3000 Hz, SNR = 3162, signal levels = 8. Find Nyquist capacity and Shannon capacity.
SOLVED
▼
Nyquist (L=8=2³, log₂8=3): N = 2×3000×3 = 18,000 bps = 18 Kbps
Shannon: N = 3000 × log₂(1+3162) = 3000 × log₂(3163)
log₂(3163): 2¹¹=2048, 2¹²=4096. 3163 is between, closer to 4096. log₂(3163) ≈ 11.63
N = 3000 × 11.63 ≈ 34,890 bps ≈ 34.9 Kbps
Answer
Nyquist = 18 Kbps | Shannon ≈ 34.9 Kbps | Limiting factor = Nyquist (lower)
BR5
Channel B = 4000 Hz, L = 4 levels, SNR = 63. Find Nyquist and Shannon. Which limits?
SOLVED
▼
Nyquist (L=4, log₂4=2): N = 2×4000×2 = 16,000 bps = 16 Kbps
Shannon: N = 4000 × log₂(1+63) = 4000 × log₂(64) = 4000 × 6 = 24,000 bps = 24 Kbps
Answer
Nyquist (16 Kbps) is the limiting factor. Actual max = 16 Kbps.
BR6
A signal has S = 1000 baud and carries 2 bits per baud. What is the bit rate?
PRACTICE
▼
N = S × bits per baud = 1000 × 2 = 2000 bps
Alternatively: L = 2² = 4 levels. N = S × log₂(L) = 1000 × 2 = 2000 bps ✓
BR7
For AMI encoding with bit rate 2 Mbps, what is the baud rate? How does it compare with NRZ?
PRACTICE
▼
AMI: c=1, r=1 → S = cN/r = 2 Mbps / 1 = 2 Mbaud
NRZ worst case: S = N = 2 Mbaud (same baud rate)
AMI and NRZ have the same baud rate, but AMI uses 3 voltage levels (+, 0, −), giving better error detection (a single-level violation is an error) and no DC component.

Distance Vector Routing — Table Problems
4 problems
DV1
Bellman-Ford update rule — how to fill a distance vector table
CONCEPT
▼
Formula: D(x, y) = min over all neighbors v { cost(x→v) + D(v, y) }
Initialization: Each router knows only its direct neighbors. Cost to self = 0. Cost to direct neighbor = link weight. Cost to all others = ∞.
Update: When router X receives table from neighbor V, for each destination Y: new_cost = cost(X→V) + D(V,Y). If new_cost < current D(X,Y), update the table and next hop.
Convergence: Keep exchanging tables until no more updates happen.

DV4
Count-to-infinity problem — what happens when A-B link fails in A-B-C topology?
TRICK
▼
Setup: A-B cost=1, B-C cost=1. Optimal: A→C via B = cost 2. B→C = cost 1.
A-B link fails. A sets D(A,B)=∞, D(A,C)=∞.
B still has table showing A can reach C with cost 2. B computes: D(B,C) via A = 1 + 2 = 3. B updates to cost 3 via A. (But A→C is already broken!)
A receives B's update. A sees B says it can reach C at cost 3. A sets D(A,C) = 1+3 = 4. Now B gets A's update and sets its cost to 5... and so on → count-to-infinity loop.
Solutions: Split horizon (don't advertise a route back to the neighbor you learned it from). Poison reverse (advertise ∞ back). Route poisoning + holddown timers. These limit but don't fully solve the problem.
